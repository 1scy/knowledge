# 后端问题

* * *

## [问题描述模板](#问题描述模板)

```markdown
背景(可选)

1. 阐述问题出现的背景信息，例如系统、软件版本、进行过哪些可疑操作

问题现象

1. 描述问题发生时的现象，例如出现CPU突然飙升、内存突然飙升，日志出现某些关键字
2. 描述问题造成的某些影响，例如出现OOM、栈溢出、用户访问前端卡顿等

猜测原因(可选)

1. 出现问题时，以第一感觉为准，凭借以往的经验以及发生的现象猜测出现问题的原因，此处记录这条信息
   有助于日后出现问题时帮助更好地排查问题，可以培养对问题的敏锐度

定位问题(可选)

1. 逐步展示定位问题的步骤，详细描述排查问题的过程
2. 对定位问题所使用的手段进行描写，例如开源库出现问题就
   去Github仓库的issue查找相关问题、使用Arthas对应用进行调用分析

原因分析(可选)

1. 对问题原因进行分析，可能是业务代码方面问题，例如Redisson中对
   锁的释放时间进行了设置导致watchdog失效、定时任务的扫表操作有慢查询导致数据库连接打满而挂掉
2. 对问题原因进行分析，可能是环境及第三方库问题，例如openjdk的slim和alpine版本的镜像
   会缺少字体而导致镜像操作Excel时异常、hutool库json工具不支持循环依赖而栈溢出问题
3. 自己难以分析的，也可引用大佬的问题分析链接

问题原因

1. 总结问题原因

解决方法

1. 解决问题的方法以及后续的避免措施
```

